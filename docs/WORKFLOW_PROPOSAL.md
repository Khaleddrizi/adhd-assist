# مقترح سير العمل والواجهات — ADHD Assist

## 1. التوافق مع الكود الحالي

### ✅ ما يتوافق

| المكوّن | الحالة |
|---------|--------|
| **API Auth** | `POST /api/auth/login` و `POST /api/auth/register` — يدعمان Orthophoniste و Parent |
| **سير عمل الطبيب** | إضافة مريض عبر `POST /api/specialists/patients` (name, age, diagnostic, parent_email) |
| **سير عمل ولي الأمر** | جلب أطفاله عبر `GET /api/parents/children` |
| **مربط Alexa** | كل مريض له `alexa_code`؛ الطفل يقول الكود في Alexa فيربط حساب Alexa بالمريض |
| **Models** | Specialist, Parent, Patient, QuizSession — الهيكل يدعم السيناريو كاملًا |

### ⚠️ ما يحتاج تعديل

1. **تسجيل الولي من طرف الطبيب:** حالياً الولي يسجّل نفسه (Register). إذا تريد الطبيب يسجّل الولي ووليدو معاً، نحتاج:
   - Endpoint جديد: مثلاً `POST /api/specialists/register-family` (parent_email, parent_password, parent_name, child_name, child_age, diagnostic)
   - أو نبقى بالسير الحالي: الولي يسجّل أولاً، ثم الطبيب يضيف الولد بريد الولي.

2. **لوحة التجربة (Dashboard 5003):** إذا لم تعد تحتاجها، يمكن إيقاف تشغيلها أو حذفها من `run_all.py`.

3. **مشكلة توجيه الطبيب:** الطبيب بعد Login يُوجَّه إلى `/orthophoniste`. إذا لم يصل للواجهة، الأسباب المحتملة:
   - خطأ بناء في صفحة orthophoniste
   - Backend (port 5004) غير شغّال
   - مشكلة في `router.push` أو AuthGuard

## 2. سير العمل المقترح (Workflow)

### أ) الصفحة الرئيسية (Home)
- عرض عام للمنتج
- أزرار: Se connecter | S'inscrire
- بدون مسارات خاصة بالداشبورد القديم

### ب) التسجيل (Register)
- اختيار نوع الحساب: **Orthophoniste** أو **Parent**
- Orthophoniste: إنشاء حساب طبيب
- Parent: إنشاء حساب ولي أمر (لاحقاً الطبيب يربطه بالطفل عبر البريد)

### ج) تسجيل الدخول (Login)
- Orthophoniste → `/orthophoniste` (واجهة الطبيب)
- Parent → `/dashboard` أو `/parent` (واجهة ولي الأمر)

### د) واجهة الطبيب (Orthophoniste)
- عرض قائمة المرضى
- إضافة مريض:
  - اسم الطفل، العمر، التشخيص
  - إيميل الولي (يفترض أن الولي مسجّل مسبقاً)
- لكل مريض: عرض **alexa_code** (ليستخدمه الطفل مع Alexa)
- خيار: "إنشاء ولي أمر + طفل" في خطوة واحدة (إذا أضفنا الـ endpoint)

### هـ) واجهة ولي الأمر (Parent / Dashboard)
- عرض قائمة أطفاله
- لكل طفل: الاسم، الكود، آخر جلسة، إحصائيات بسيطة
- لا حاجة لإحصائيات معقدة أو لوحة تجربة قديمة

### و) ربط الطفل مع Alexa
1. الطبيب يسجّل الولي (أو الولي يسجّل نفسه)
2. الطبيب يضيف الطفل ويربطه بإيميل الولي
3. الطفل يفتح مهارة Alexa ويقول الكود (مثلاً "ABCD1234")
4. Alexa تربط `alexa_user_id` بالمريض
5. جلسات الكويز تُسجَّل على المريض
6. الولي يرى تقدم طفله من واجهته

### ز) تجربة السكيل على Amazon
- أنت مسجّل كمريض (child)
- الطبيب أوصل لك الـ alexa_code
- تفتح السكيل على جهاز Echo أو محاكي، تقول الكود
- تُجرى جلسة Quiz وتُسجّل النتائج
- الطبيب و/أو ولي الأمر يرى النتائج من الويب

## 4. خيار تسجيل العائلة من الطبيب

إذا أردت الطبيب يسجّل الولي والطفل معاً دون أن يسجّل الولي نفسه:

```python
# POST /api/specialists/register-family
# Body: { parent_email, parent_password, parent_name, child_name, child_age, diagnostic }
# 1. إنشاء Parent (إذا لم يوجد)
# 2. إنشاء Patient مربوط بالطبيب وولي الأمر
# 3. إرجاع المريض مع alexa_code
```

هذا يبسط السير: خطوة واحدة من الطبيب بدل خطوتين.

## 5. الخطوات المقترحة للمتابعة

1. **إصلاح تسجيل دخول الطبيب** — التأكد من أن Login يوجّه فعلياً إلى `/orthophoniste` بدون أخطاء بناء.
2. **إزالة/إيقاف Dashboard 5003** إن لم تعد تحتاجه.
3. **توحيد التسجيل:** إما الإبقاء على سير "الولي يسجّل أولاً" أو إضافة `register-family` للطبيب.
4. **توثيق الـ alexa_code** — التأكد أن الطبيب وولي الأمر يريان الكود بوضوح ويعرفان كيف يستخدمه الطفل.
5. **تجربة السكيل على Amazon** — نشر السكيل، ربط المريض بالكود، وتشغيل جلسة Quiz للتأكد أن البيانات تصل للويب.
